# Security Policy

## Supported Projects

Computer TB Lab maintains security practices for active projects under this organization.

Project-specific support status may be defined within each repository.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue.

Instead, report it privately through one of the following methods:

- GitHub Security Advisories
- Direct contact with the repository maintainers

---

## Security Principles

Computer TB Lab follows these principles:

- Private-first repository management
- No secret or credential exposure
- Review-first development workflow
- AI-assisted development with human verification
- Least-privilege access control
- Two-factor authentication enforcement

---

## Sensitive Information Handling

The following information must never be committed to public repositories:

- API keys
- Access tokens
- Connection strings
- Deployment settings
- Customer data
- Internal infrastructure information

---

## AI-assisted Development Policy

AI-generated code and specifications must be reviewed before production use.

Automated tools including AI assistants must not expose secrets, customer data, or internal credentials.

---

## Contact

For security-related inquiries, contact the maintainers of Computer TB Lab.
